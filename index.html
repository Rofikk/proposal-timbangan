Define: Masalah yang Akan Kami Perbaiki

Bagian Define menjelaskan kondisi saat ini (AS-IS) dan kenapa perlu diperbaiki. Tujuannya supaya Bapak/Ibu punya gambaran yang sama: di mana sumber lambatnya proses dan di mana celah risiko.

Risiko salah tulis Proses lambat Sulit audit

Alur Proses Saat Ini (Manual)

  1. Pencatatan di kertas setelah timbang
    Hasil timbangan ditulis manual. Di titik ini, angka sangat bergantung pada ketelitian manusia.
  2. Kertas diserahkan ke admin untuk “dirupiahkan”
    Terjadi serah-terima fisik. Kalau kertas terlambat, hilang, atau rusak, proses ikut terganggu.
  3. Admin memeriksa barang untuk verifikasi
    Verifikasi dilakukan terpisah dari proses timbang. Risiko mismatch data (barang vs catatan) jadi lebih tinggi.
  4. Admin menghitung total transaksi harian untuk pembayaran
    Rekap dan penjumlahan manual memakan waktu dan rawan salah hitung, terutama saat transaksi ramai.
  5. Supervisor merekap transaksi periode berjalan
    Karena data tersebar (kertas/rekap manual), supervisor butuh waktu ekstra untuk memastikan data “beres”.
  6. Supervisor merekap distribusi ke gudang pusat
    Traceability (jejak dari timbang sampai gudang) lemah, sehingga selisih lebih sulit ditelusuri sumbernya.

Apa Risiko Utamanya?

  • Salah tulis angka timbang → berdampak langsung ke nilai pembayaran.
  • Kertas hilang/robek → transaksi tertahan / butuh input ulang.
  • Data tidak punya jejak digital → sulit audit ketika ada komplain/selisih.

Kenapa Proses Terasa Lambat?

  • Serah-terima kertas membuat antrean (bottleneck).
  • Verifikasi dilakukan terpisah, jadi “bolak-balik”.
  • Rekap harian manual makan waktu saat jam tutup.

Apa Dampak ke Manajemen?

  • Sulit melihat angka real-time (harus tunggu rekap).
  • Sulit menelusuri selisih timbang vs gudang pusat.
  • Transparansi dan akuntabilitas terbatas.

Visual Singkat: Titik Masalah

Kesimpulan Define: masalah utama bukan hanya “pencatatan”, tetapi rantai proses yang bergantung pada kertas. Ini membuat proses lambat, rawan salah, dan sulit ditelusuri saat ada selisih atau audit.
